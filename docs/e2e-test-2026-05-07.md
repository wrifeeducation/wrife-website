## Bugs Found

### 🐛 Bug #1 — `ANTHROPIC_API_KEY` not configured → mock AI assessment
- **Where:** PWP Studio `assess-formula` Edge Function v3
- **Impact:** All formula submissions return `overall_score: 70` regardless of sentence quality. No real AI feedback shown to pupil.
- **Fix:** Set `ANTHROPIC_API_KEY` secret on the Platform Supabase project via CLI: `supabase secrets set ANTHROPIC_API_KEY=<key> --project-ref gzmgjkbtsvezfclmreru`
- **Severity:** High (core feature non-functional in production)

---

### 🐛 Bug #2 — IP login form label mismatch ("YOUR FIRST NAME" vs username)
- **Where:** Interactive Practice `/login` page — the username field is labelled "YOUR FIRST NAME"
- **Impact:** Pupils are likely to type their first name (e.g. "Amadeo") instead of their username (e.g. "amab04"), causing login to fail
- **Fix:** Change the label in the IP app login form from "YOUR FIRST NAME" to "YOUR USERNAME"
- **File:** `src/pages/AuthPage.tsx` or equivalent login component in InteractivePracticeApp
- **Severity:** High (causes login failures for pupils)

---

### 🐛 Bug #3 — World Map sidebar stats not re-hydrating after lesson completion
- **Where:** Interactive Practice World Map page — sidebar XP and lessons count
- **Symptom:** After returning to World Map from a completed lesson, "TOTAL XP" shows 0 and "LESSONS DONE" shows 0. The lesson node itself renders correctly (purple completed circle). DAY STREAK updates correctly (shows 1), so streak query runs fine.
- **Root cause:** Zustand `gameStore` initialises XP/lessons to 0 and doesn't re-query Supabase when the World Map page mounts post-navigation.
- **Fix:** In `WorldMapPage.tsx` (or the `useProgress` hook), add a `useEffect` that refetches `pupil_progress` and updates `gameStore` when the component mounts. Ensure `lessonCount` and `totalXP` are derived from DB state, not just accumulated Zustand state.
- **Severity:** Medium (data is correct in DB; UI display only)

---

### 🐛 Bug #4 — "??" avatar on World Map sidebar
- **Where:** Interactive Practice World Map — pupil profile avatar
- **Symptom:** Shows "??" where initials should appear; "Pupil / Level 1" text renders but display name is missing
- **Root cause:** Profile fetch on World Map mount is likely returning `null` for `display_name` or failing silently
- **Fix:** Check what field `WorldMapPage.tsx` reads for the pupil display name; verify `pupil-login` Edge Function returns a `display_name` or `first_name` field and that the Zustand store receives it correctly
- **Severity:** Low (cosmetic)

## Design Gap

### ⚠️ IP lesson completion ≠ assignment submission
- **Observation:** Amadeo completed IP Lesson 1 (confirmed in `pupil_progress` and shown in the IP tab with 1/61), but the Assignments tab still shows "0/25 submitted" for the "Developing Awareness of Personal Stories" assignment.
- **Current design:** `pupil_progress` (lesson completion) and `class_ip_assignment_submissions` (formal assignment submission) are separate tables. Completing a lesson does **not** automatically create a submission record.
- **Question for product decision:** Should completing an assigned lesson automatically count as submitting it? If yes, the IP app needs to INSERT into `class_ip_assignment_submissions` when `pupil_progress` is written for a lesson that has an active assignment. If no, pupils need a separate "submit" action after completing the lesson.
- **Recommendation:** Auto-submit is the better UX for primary school pupils — explicit submission steps add friction and confusion.

## SSO Verification

The SSO link chain from wrife.co.uk pupil dashboard → PWP Studio and → Interactive Practice was verified as working end-to-end:

1. Pupil logs in at wrife.co.uk with class code + username + PIN
2. Dashboard renders with PWP Studio and Interactive Practice entry cards
3. Clicking either card generates a signed SSO URL (`buildSSOUrl()` → `?access_token=...&refresh_token=...`)
4. Target app receives the tokens, calls `supabase.auth.setSession()`, and lands the pupil in their session
5. Both apps confirmed pupil identity correctly (Amadeo B, `d4474871-...`)
